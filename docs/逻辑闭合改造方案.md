# 地质勘探报告「逻辑闭合」诊断 + 改造方案

> 本文档为 2026-06-08 针对报告逻辑链「不闭合」问题的诊断与改造记录。改造已实施，见文末「实施状态」。

## Context（为什么做这件事）

当前生成的报告读起来「逻辑链不闭合」。一份找矿评价报告的理想闭环是：

```
目标(找什么矿·为什么这个区·用什么方法)
   → 证据(各章节数据·每章对成矿/施工的指示)
   → 综合研判(靶区圈定 + A-D 置信，且二者自洽)
   → 结论与建议(回扣目标，指向最薄弱环节)
```

逐文件核查后，确认链路在四处断开。下面给出每一处的现象 → 根因 → 改造方案。

---

## 断点 1：计算闭合 —— 靶区评级与综合置信是两条互不相干的路径

**现象**：报告里出现两个「置信」结论——靶区表的 A/B/C/D 评级、综合置信评价章的 A/B/C/D 等级——但两者可以互相矛盾，读者无法理解二者关系。

**根因**：`build_target_zone_figure(location)` 只接收 location，与 `evaluate_confidence` 是两次独立调用；靶区分级仅依据 geo-exploration broker + 子系统可用性计数，无视证据章节与目标矿种。

**改造**：新增 `synthesis.evaluate_synthesis()`，**单次 claude -p 同时产出**综合置信等级与靶区/有利地段研判，共用同一证据上下文，二者自洽；`web/app.py` 改为调用该统一入口。

---

## 断点 2：靶区坐标与「评分理由」在无深部数据时是凭空生成的

**现象**：无 geo-exploration 数据时，靶区表仍给出具体坐标与「多源异常高度套合」套话，无证据来源。

**改造**（采用方案 A）：无深部靶区时**不再输出精确坐标**，改为「找矿有利地段/远景区」定性条目（方位/依据/建议查证手段），底图只画研究区框，并明确标注「未经深部探测验证，坐标待野外厘定」。`synthesis._build_figure` 以 `mode=areas|targets` 区分。

---

## 断点 3：维度对齐 —— 一半章节进不了最终研判，等级未强制自洽

**改造**：
1. 置信维度由 6 个扩为 7 个，新增「勘探可行性与施工条件」，汇总地形/水文/气候/交通/矿业权各章的 `exploration_impact`；prompt 要求对每章给出 `chapter_disposition`（去向），杜绝悬空章节。
2. 新增 `synthesis._cap_grade()`：多数维度为低/缺、或缺少子系统本地实证（仅网络资料）时，综合等级回拉至不高于 C，并在 summary 注明回拉原因。

---

## 断点 4：可追溯性 —— 结论不引章节，全文删源使「可信度」维度无法核验

**改造**：
1. `search_engine` 确定性追踪每章证据来源层级（子系统本地实证 / 直连API / 网络检索），写入 `SearchResult.evidence_level`，报告每章末以小字标注。
2. `confidence_prompt.j2` 传入带**章节号**的证据，要求 summary / dimensions.note / 靶区 reason 回引「见第X章」。
3. `recommendation` 要求针对最薄弱维度与最低置信靶区/地段给出具体动作。

---

## 断点 5：结构闭合 —— 缺前言与结论先行

**改造**：`report_builder._add_front_matter()` 在封面后、第一章前新增：
- **前言**：工作目标 / 数据构成（按来源层级统计章节数）/ 技术路线。
- **执行摘要（结论先行）**：综合置信等级 + 结论摘要 + 首选靶区/最有利方向 + 下一步建议，与末章呼应。

---

## 改动的关键文件

- `reporter/synthesis.py` — 统一研判 `evaluate_synthesis`、等级自洽 `_cap_grade`、`mode` 分流 `_build_figure`、确定性兜底；保留旧接口兼容包装。
- `reporter/search_engine.py` — 证据来源层级追踪（`_fetch_one_category` 返回 levels，写入 `evidence_level`）。
- `reporter/categories.py` — `SearchResult.evidence_level` 字段。
- `reporter/report_builder.py` — 前言+执行摘要、来源层级小字、靶区/有利地段双模渲染。
- `reporter/pptx_builder.py` — 靶区/有利地段双模幻灯片。
- `templates/confidence_prompt.j2` — 统一输出（置信+靶区/有利地段）、7 维、章节去向、章节回引、双模。
- `web/app.py` — `_do_synthesis` 改用 `evaluate_synthesis` 单次统一研判。

## 验证方法（端到端）

1. 有深部数据：靶区等级与综合置信自洽（自洽校验回拉），靶区理由回引章节。
2. 无深部数据：靶区章为定性「有利地段」、无伪精确坐标；多数维度低/缺时综合等级 ≤ C。
3. 检查 .docx：开头有前言+执行摘要，结论与末章一致；每章末有来源层级；建议指向最弱项。
