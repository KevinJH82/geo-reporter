# geo-reporter 子系统消费数据重构方案

## Context（为什么做这次重构）

geo-reporter 当前对接 8+ 个兄弟子系统，但「哪个章节消费哪个子系统」的映射与融合逻辑是**分散硬编码在 `data_sources.py:fetch_direct` 与 `collect_subsystem_figures` 的 if/elif 分支里**，缺少统一规划，导致：

1. **geo-geophys 子系统整体漏接** —— `_geo_geophys_figures()`（`data_sources.py:419`）已写好却从未被任何分支调用；geophysics 章节至今只用 data-colle 的二手物探文本/图件，专业位场处理成果（化极/解析信号/倾斜角/欧拉磁源深度/plain_summary）完全进不了报告。
2. **geo-model3d 图件漏接** —— `_geo_model3d_figures()`（`data_sources.py:312`）已写好，但 `collect_subsystem_figures` 的 geology 分支只调 geo-stru，三维成矿预测的深度剖面/切片图进不了报告。
3. **深部探测靶区错位** —— geo-exploration 深部靶区被塞进「遥感影像分析」章节（`fetch_direct` 的 `remote_sensing` 分支），地学语境不对；而它本应只服务靶区推荐+置信研判。
4. **insar 对接方式不一致** —— `fetch_insar_local` 手工遍历目录读 json，未走已有的 `commons.insar_broker.find_insar_for_bbox()`。
5. **证据层级靠字符串嗅探** —— `search_engine.py:25` 的 `_SUBSYSTEM_MARKERS` 通过在文本里搜「本地实证/优先于Web」字样来反推来源层级，脆弱且易错标。
6. **大量已产出数据未消费** —— deposits_broker（已知矿点）、drill_broker（钻探布孔/反馈）、geochem_public_broker（公开化探）、geophys 欧拉磁源点位、geochem 元素阈值表。

**目标产出**：引入**声明式「章节—子系统」消费契约表 + 标准 Provider 接口**，集中管理所有消费与融合优先级；同时修正上述错配、补齐 geo-geophys/deposits/drill/geochem 四类新消费。报告仍保持现有 10 章结构，仅重新规划每章消费的数据。

---

## 决策基线（已与用户确认）

- 重构方式：**声明式消费契约表**（统一映射 + 标准 provider），非渐进修补。
- 章节结构：**保持现有 10 章不变**，只重新分配数据消费。
- 新增消费：**geo-geophys 深化 + deposits 已知矿点 + drill 钻探闭环 + geochem 深化**，全部纳入。

---

## 重新规划后的「章节 ↔ 子系统」消费映射

层级标记：**L**=子系统本地实证（最高），**API**=直连权威 API，**Web**=网络检索兜底。`→` 表示优先级回退链。

| 章节 | 文本消费（按优先级） | 图件消费 | 变化 |
|---|---|---|---|
| **geology** 地质与矿产 | geo-stru构造(L) + geo-model3d建模小节(L) + **deposits已知矿点(L)**〔新〕 + data-colle地质(L) + Macrostrat(API) | geo-stru(L) + **geo-model3d深度剖面/切片(L)**〔新接〕 → 兜底 Macrostrat | +deposits 文本、+model3d 图件 |
| **geography** 地理地形 | geo-stru地形(L) + data-colle地理(L) + SRTM(API) | — | 不变 |
| **geophysics** 地球物理 | **geo-geophys位场+欧拉磁源深度+plain_summary(L)**〔新〕 → data-colle物探(L,兜底) | **geo-geophys图件(L)**〔新接〕 → data-colle(兜底) | 主源改为 geo-geophys |
| **geochemistry** 地球化学 | geo-geochem异常+浓集中心+**元素阈值表(L)**〔深化〕 + **geochem_public公开化探(L)**〔新〕 + data-colle(L) | geo-geochem(L) | +阈值表 +公开化探 |
| **remote_sensing** 遥感 | geo-analyser蚀变(L)〔**移除深部靶区**〕 | geo-analyser(L)〔**移除深部靶区图**〕 | 深部靶区移出 |
| **insar_deformation** | geo-insar(L)〔**改走 insar_broker**〕 | （可选）形变证据图 | 统一 broker |
| hydrology / climate | OSM(API) / Open-Meteo(API) | — | 不变 |
| infrastructure / mining_rights | Web | — | 不变 |

**靶区推荐（第十章）+ 置信评价（第十一章）** 的研判证据（`synthesis.py`）：

| 证据 | 子系统 | 用途 | 变化 |
|---|---|---|---|
| 深部探测靶区（坐标+潜力值） | geo-exploration(L) | 驱动靶区推荐图、置信研判 | 从遥感章节迁来，专供研判 |
| **已知矿点套合** | deposits(L) | 靶区与已知矿点空间关系、置信佐证 | 新增 |
| **钻探布孔+反馈（见矿/无矿）** | drill(L) | 靶区验证闭环、布孔建议 | 新增（geo-drill 无产出时降级跳过） |
| 子系统可用性 | 全部 | `_cap_grade` 置信回拉 | 扩展纳入 geophys/geochem/model3d/deposits |

## 实施步骤

### 1. 新建 `reporter/consumption.py` —— 声明式消费契约 + 融合层（核心）

定义统一抽象，取代散落的 if/elif：

```python
# 来源层级（显式声明，取代 search_engine 的字符串嗅探）
LEVEL_LOCAL = "子系统本地实证"
LEVEL_API   = "直连API"
LEVEL_WEB   = "网络检索"

@dataclass
class ProviderResult:
    texts: List[str]            # 注入 LLM 的原始文本
    figures: List[dict]         # {path, caption, source}
    level: str                  # LEVEL_LOCAL / LEVEL_API
    ok: bool                    # 是否取到数据（用于回退判断）

# 每个 provider 包装一个现有 data_sources 函数，统一签名 (bbox, lat, lon) -> ProviderResult
# 章节消费契约：每章一组 text/figure provider，按列表顺序即优先级
CHAPTER_CONTRACT = {
  "geology":     ChapterSpec(text=[stru, model3d_summary, deposits, datacolle("geology"), macrostrat],
                             figures=[stru_figs, model3d_figs], figures_fallback=[macrostrat_map]),
  "geophysics":  ChapterSpec(text=[geophys, datacolle("geophysics")],   # geophys 优先，data-colle 兜底
                             figures=[geophys_figs], figures_fallback=[datacolle_figs]),
  "geochemistry":ChapterSpec(text=[geochem, geochem_public, datacolle("geochemistry")],
                             figures=[geochem_figs]),
  "remote_sensing":ChapterSpec(text=[alteration], figures=[alteration_figs]),
  ...
}

def consume_chapter(category_id, lat, lon, bbox) -> {"texts":[(text,level)], "figures":[...]}:
    """遍历契约表 provider，聚合文本（带层级）+ 图件；figures 为空时用 figures_fallback。"""
```

- **融合规则统一在此**：text provider 按顺序全部聚合（不互斥，本地实证+API 并列）；figure provider 主链为空才启用 fallback（如 geophys 图件缺失才用 data-colle 图件，geology 图件缺失才渲染 Macrostrat 公开地质图）。
- **层级显式携带**：每个 provider 返回的 `level` 直接进 `evidence_level`，**删除 `search_engine.py:25` 的 `_SUBSYSTEM_MARKERS` 字符串嗅探**。
- 复用现有函数作为 provider 实现：`_geo_stru_figures`、`_geo_model3d_figures`、`_geo_geophys_figures`、`_geo_geochem_figures`、`fetch_structural_local`、`geo_model3d_modeling_summary`、`fetch_geochem_summary_text`、`fetch_alteration_local`、`fetch_datacolle_section` 等（均在 `data_sources.py`）。

### 2. `reporter/data_sources.py` —— 补齐缺失 provider 函数

新增以下函数（与现有 `fetch_*_local` 同风格，返回 `List[str]` 文本，失败/无数据返回 `[]`）：

- `fetch_geophys_text(bbox)` —— 调 `commons.geophys_broker.find_geophys_for_bbox` + `load_euler_sources`，输出 plain_summary、欧拉磁源深度中位/点位、化极解析信号要点。**复用已有路径常量 `GEO_GEOPHYS_OUTPUTS`（行413）。**
- `fetch_known_deposits_text(bbox)` —— 调 `commons.deposits_broker.find_deposits_for_bbox` + `get_points`，输出已知矿点 name/commodity/deposit_type/坐标。新增 `GEO_DEPOSITS` 路径（geo-model3d/results/<AOI>/deposits）。
- `fetch_drill_text(bbox)` —— 调 `commons.drill_broker.find_drill_for_bbox` + `get_holes`/`get_feedback`，输出计划孔位+反馈。新增 `GEO_DRILL_OUTPUTS = _ROOT + "/geo-drill/results"`；**geo-drill 当前无实际产出，函数取不到即返回 `[]`，调用方降级跳过。**
- `fetch_geochem_public_text(bbox)` —— 调 `commons.geochem_public_broker.find_public_geochem_for_bbox` + `load_public_geochem_df`，输出公开化探统计摘要。
- 扩充 `fetch_geochem_summary_text` —— 追加 `model_stats.prior_only.thresholds` 元素阈值分级表（background/weak/moderate/strong）。
- `fetch_insar_local` 重写为走 `commons.insar_broker.find_insar_for_bbox`（消费 `stats` 的 deformation_rate/coverage/n_bursts），保留旧目录遍历为兜底。

`fetch_direct` 与 `collect_subsystem_figures` **改为薄封装**，内部委托 `consumption.consume_chapter`，保留对外签名不变（`search_engine.py:17` 仍 import 它们）。

### 3. `reporter/search_engine.py` —— 用显式层级替换字符串嗅探

- `_fetch_one_category`（约行 124-210）改为接收 `consume_chapter` 返回的 `(text, level)`，直接据 `level` 累积 `evidence_level`，删除 `_SUBSYSTEM_MARKERS`/对应嗅探逻辑（行 24-25 及使用处）。
- remote_sensing 不再注入深部探测文本（已在契约表移除）。

### 4. `reporter/synthesis.py` —— 研判证据纳入 deposits/drill，扩展可用性

- `_subsystem_availability`（行 105-134）扩展 `geophys/geochem/model3d/deposits/drill` 五个键，供 `_cap_grade` 置信回拉判断更全面。
- 新增 `get_known_deposits(location)`、`get_drill_evidence(location)`，与现有 `get_prospecting_targets`（行 34）并列，注入统一研判 prompt 与 `_gather_evidence`：靶区与已知矿点的空间套合、钻探见矿/无矿反馈作为靶区分级的佐证。
- 复用 `_import_commons` 与路径常量（从 `data_sources` 导入，行 22-23 处补充 deposits/drill 路径）。

### 5. `reporter/report_builder.py` —— 靶区推荐章节渲染新证据

第十章（靶区推荐）增补：已知矿点分布小表（deposits 有数据时）、钻探布孔/反馈小表（drill 有数据时）。无数据则整段不出现（沿用现有「过滤暂缺占位」风格）。geo-model3d 深度切片图、geo-geophys 图件经契约表进入对应章节后由现有 `_add_figure` 自动渲染，无需改渲染逻辑。

## 关键设计原则（延续现有约定）

1. **不伪造证据**：drill/deposits/geochem_public 无产出 → provider 返回 `[]` → 章节静默降级，不输出占位套话（沿用 `pptx_builder._NO_DATA_MARKERS` 风格）。
2. **优先级即契约**：本地实证 > 直连API > 网络检索的顺序在 `CHAPTER_CONTRACT` 里声明，不再隐藏在 if/elif。
3. **路径常量集中**：新子系统路径继续在 `data_sources.py` 顶部基于 `_repo_root()` 推导，所有 broker 调用显式传目录（沿用 commit d683d46 的修复约定）。
4. **向后兼容**：`fetch_direct`/`collect_subsystem_figures` 对外签名不变，改造对 `search_engine`/`web/app.py` 透明。

## 验证方式（端到端）

测试 ROI 用已有真实产出的 **山东招远庙山金矿**（各子系统 results 下均有该 AOI 输出）。

1. **单元级**：写一次性脚本 `python -c` 调 `consumption.consume_chapter("geophysics", lat, lon, bbox)`，确认返回 geo-geophys 图件与位场文本、层级为「子系统本地实证」；对 geology 确认含 geo-model3d 深度切片图与 deposits 已知矿点文本。
2. **端到端**：启动 `python web/app.py`，上传招远 KML，生成报告，核对：
   - geophysics 章节出现 geo-geophys 位场图 + 欧拉磁源深度文本（而非仅 data-colle）；
   - geology 章节出现 geo-model3d 深度剖面/切片图 + 已知矿点；
   - geochemistry 出现元素阈值分级表；
   - remote_sensing **不再**出现深部探测靶区；
   - insar 数据来自 insar_broker；
   - 靶区推荐章节纳入 deposits/drill（drill 无产出时该小表不出现，且无报错）。
3. **回归**：确认每章末「证据来源层级」标注由显式 `level` 驱动正确显示，气候/水文/矿业权等未改章节输出不变。
